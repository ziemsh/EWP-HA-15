Der git- Workflow

In dieser kurzen Darstellung sollen die einzelnen git - Befehle gezeigt werden.

Dabei handelt es sich um folgende Befehle:

  • git clone
  • git add
  • git commit
  • git push
  • git pull

git clone ist der Befehl der zum Klonen der Daten vom git repository auf lokalem Speicher benötigt wird. Die Daten können nun lokal geändert werden.

Um die geänderten Daten zwischenzuspeichern wird der Befehl git add angewandt, die Zwischenspeicherung wird so vorbereitet und mit git commit vollzogen.

Letztendlich sollen die geänderten Daten wieder in das zentrale Repository hochgeladen werden. Mit git push wird dies umgesetzt.

Sollen Daten im zentralen Repository synchronisiert werden, kann dies mit git pull erfolgen.
